# Day 5 — Capstone Project (Dashboard + Login + Users List)

##  Objective
Build a small multi-page dashboard using all components created in previous days.

##  Topics Covered
- Page organization
- Mock APIs / mock data
- Reusing components
- Clean folder architecture

##  Tasks / Exercises
Create pages:
- /login
- /dashboard
- /dashboard/users
- /dashboard/profile

##  Folder Structure
/app/login/page.jsx  
/app/dashboard/page.jsx  
/app/dashboard/users/page.jsx  
/app/dashboard/profile/page.jsx  
/components/ui/*  
/data/users.js (optional mock data)  

## ▶ How to Run
npm install
npm run dev



##  Features Built
- Login UI
- Dashboard widgets
- Users table (mocked data)
- Reusable UI components


## Notes / Learnings
- Component reuse speeds up development.
- Folder structure matters for scaling.
- Mock data helps when backend is not available.
